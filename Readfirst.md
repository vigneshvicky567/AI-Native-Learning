# Lexi AI - Project Overview

## About the Project
Lexi AI is an AI-native learning website dashboard inspired by a modern assistant interface. It acts as a Computer Science tutor that teaches algorithms and data structures visually. The application is built using React, Vite, Tailwind CSS, and integrates directly with the Gemini API to provide step-by-step graph tutorials, interactive HTML widgets, and learning checkpoints.

## What Works
- **Chat Interface**: Users can interact with the Gemini API.
- **Visual Tutorials**: The AI can return JSON-based step-by-step graph tutorials or custom interactive HTML widgets (`<visual>` tags).
- **Checkpoints**: The AI can generate learning checkpoints which are displayed in a sidebar and can be toggled by the user.
- **Code Editor**: A built-in code editor for users to write and "run" code (currently simulated by the AI).
- **UI/UX**: Dark mode toggle, responsive design, and a polished landing page.

## What Doesn't Work (Needs Improvement)
- **Security**: The application currently calls the Gemini API directly from the client side, exposing the API key.
- **Code Execution**: The "Run Code" feature currently asks Gemini to simulate the output rather than executing it in a secure sandbox.
- **Persistence**: Chat history and checkpoints are stored in local React state and are lost upon page refresh. There is no database integration yet.
- **Backend**: A FastAPI backend is planned (as detailed in `FASTAPI_ARCHITECTURE.md`) to handle API calls securely, execute code, and manage database storage.

## File Structure in Brief

### Root Files
- `package.json`: Project dependencies and scripts.
- `metadata.json`: Application metadata (name, description).
- `FASTAPI_ARCHITECTURE.md`: Documentation outlining the planned FastAPI backend architecture to replace client-side logic.
- `vite.config.ts`: Vite bundler configuration.
- `tsconfig.json`: TypeScript configuration.

### `src/` Directory
- `main.tsx`: The entry point of the React application.
- `App.tsx`: The main application component. It manages global state (view, messages, checkpoints, dark mode) and handles the direct communication with the Gemini API.
- `index.css`: Global CSS, including Tailwind imports and custom styles.

### `src/components/` Directory
- `LandingPage.tsx`: The initial screen the user sees before starting a chat.
- `ChatView.tsx`: The main chat interface containing the message history, input area, and toggles for the code editor and checklist.
- `ChecklistSidebar.tsx`: A sidebar that displays the learning checkpoints generated by the AI.
- `CodeEditor.tsx`: The Monaco-based code editor component.
- `Header.tsx`: The top navigation bar within the chat view.
- `InputArea.tsx`: The component where users type their messages.
- `Sidebar.tsx`: The main application sidebar.
- `TutorView.tsx`: Responsible for rendering the structured JSON data from the AI (graphs, code, quizzes).
- `GraphVisual.tsx`: Renders the node/edge graph visualizations.
- `CodeBlock.tsx`: Renders syntax-highlighted code snippets.
- `BlockRenderer.tsx`: Helper component to render different types of content blocks.
- `Cards.tsx`: Reusable card UI components.
- `Greeting.tsx`: Displays a greeting message.

### `src/components/ui/` Directory
- Contains reusable, lower-level UI components such as `button.tsx`, `textarea.tsx`, `ai-prompt-box.tsx`, `spinner.tsx`, etc., many of which are likely based on shadcn/ui or similar libraries.
